WITH fully_enabled AS (
  SELECT DISTINCT CAST(ff_orgs.org_id AS BIGINT) AS org_id
  FROM (
    SELECT EXPLODE(FROM_JSON(fferc.values, 'ARRAY<BIGINT>')) AS org_id
    FROM datamodel_launchdarkly_bronze.feature_flag_environment_rule_clause fferc
    WHERE fferc.feature_flag_key = 'next-gen-coaching-feature-enabled'
      AND fferc.attribute = 'orgId' AND fferc.context_kind = 'user'
      AND fferc.date = (SELECT MAX(date) FROM ...)
  ) ff_orgs
  INNER JOIN datamodel_core.dim_organizations do
    ON ff_orgs.org_id = do.org_id ...
  WHERE do.internal_type = 0
),
segment_org_ids AS ( ... ),
segment_emails AS ( ... ),
email_user_orgs AS ( ... ),
email_orgs AS ( ... ),
beta_orgs AS (
  SELECT org_id FROM fully_enabled
  UNION SELECT org_id FROM segment_org_ids
  UNION SELECT org_id FROM email_orgs
)

High-level summary

The guided coaching experience is working. Coaches are adopting sessions, using Coach Now from the inbox, and the shift away from one-offs is clear at the user level.

What’s working

  1. Coach behavior has shifted — 81% → 91% session adoption is the clearest success signal
  2. Coach Now is a reliable entry point — 12–15% weekly usage, 40–53% completion
  3. Volume is growing — +24% per-day completed coaching Dec → Mar

What needs attention

  1. Fraley & Schilling — nearly 100% one-offs, pulls down volume metrics
  2. Breakthru Beverage — swings 39–65% session share
  3. Data gaps — AI Talk Track & Switch-back not in Databricks
  4. Switch-back ~27% — 1 in 4 users tries old UI at least once
Why user-level metrics differ from session-level: % users measures how many people adopted the behavior (binary per person). % sessions is volume-weighted — one coach doing 100 one-offs counts 100 times. This is why coach adoption shows +10 pp (81% → 91%) while session share is a flatter ~65%. The few remaining one-off-heavy coaches generate outsized volume that anchors the session metric down.

Methodology

  • Active coach pool (denominator): 320 unique coach_ids from coachingdb_shards.coaching_sessions who completed ≥1 session (type 1 or 2) between Feb 2 – Mar 15, 2026 in the 6 fully enabled orgs.
  • Email mapping: Mixpanel distinct_id = email address; coach_id = numeric user ID. Joined via datamodel_platform.dim_users (user_id ↔ email) to match Coach Now / v2 completion events to the active coach pool.
  • Coach Now source filter: source = 'aggregated_inbox_coach_driver_button' on fleet_safety_coaching_sessions_click_begin_sessions.
  • Switch-back: Mixpanel switchToOldUI unique users by week (5 orgs, excl. Dyno Nobel) divided by Databricks active coaches for the same 5 orgs.
  • AI Talk Track: Mixpanel ai_talk_track_tab_clicked unique users by week (5 orgs, excl. Dyno Nobel) divided by Databricks active coaches for the same 5 orgs.
